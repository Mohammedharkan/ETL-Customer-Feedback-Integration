# ETL Customer Feedback Integration Project

## Project Overview

This project focuses on integrating customer feedback data from multiple sources into a centralized SQL database for analysis and reporting.

The project was completed in two parts. Part 1 focused on database design, ERD creation, SQL table development, and CSV data integration. Part 2 expanded the project by integrating JSON and XML data sources using Python and SQL.

- Python
- Jupyter Notebook
- CSV
- JSON
- XML
- Database Design
- ETL Processes

- Data Integration

## Data Sources

The project used multiple customer feedback sources:

- CSV customer survey data
- JSON web feedback data
- XML external reviews data
- SQL database tables

## What I Did

- Designed a relational database structure.
- Created an ERD to show relationships between customers, products, and reviews.
- Extracted data from CSV, JSON, and XML files.
- Cleaned, validated, and transformed the data.
- Loaded the transformed data into SQL database tables.
- Used SQL queries to analyze customer feedback.
- Identified product ratings, customer feedback trends, and recurring issues.
- Prepared reports and documentation for the ETL process.

## ETL Process

### Extract

Data was collected from CSV, JSON, and XML files.

### Transform

The data was cleaned, standardized, validated, and prepared for analysis.

### Load

The cleaned data was loaded into structured SQL database tables.

## Business Value

This project helps organizations combine customer feedback from different sources into one reliable database. This supports better reporting, customer satisfaction analysis, product improvement, and data-driven decision-making.
